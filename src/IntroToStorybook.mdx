# Introduction

Repo: [https://github.com/shalanah/storybook-playground](https://github.com/shalanah/storybook-playground)

## Pros

- Clean-room environment for building components (good patterns)
- [Fast](https://storybook.js.org/blog/storybook-9/)
- Shared documentation

## Live examples

- [Drei](https://drei.pmnd.rs/?path=/docs/staging-accumulativeshadows--docs)
- [Grafana Labs](https://developers.grafana.com/)
- [VSCode Webview UI Toolkit](https://microsoft.github.io/vscode-webview-ui-toolkit)

## Cons

- Initial setup time (+ keep up time)
- Learning curve for new users
- Best with bigger teams or complex projects
- May over complicate simple components

## Agenda

- File structure
- Create stories
  - Toolbar + addons
  - Tags + docs
  - Layouts
  - Tests
- Testing

## Conclusion

Add Storybook if your team is excited about Storybook.

There is overhead but in professional long-term projects or projects with bespoke or complex UIs, the benefits outweigh the costs.

Start small only ie: presentational new components - you don't even need to mandate it!

If you find new sweet ways to use Storybook, I'd love to hear about it!
